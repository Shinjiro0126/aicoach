タグライン「あなたの目標は、この端末から出ない。」を軸に展開する(ブランド定義は `docs/brand.md`)。

> 注意: App Store Connect への反映は手作業。貼り付け後、Connect 側の文字数カウンタでも超過していないことを確認すること。

## App Privacy ラベル設定ガイド

App Store Connect → 対象アプリ → 「アプリのプライバシー」で以下の通り申告する。
実装根拠: `src/lib/observability/sentry.ts`(Sentry、`sendDefaultPii: false`・コーチAPIペイロード除去済み)、`src/lib/analytics/posthog.ts`(PostHog、定義済みイベント名のみ・オートキャプチャ/セッションリプレイ無効・distinct id は匿名の端末生成ID)。

### 手順

1. **「データ収集」→「はい、このアプリからデータを収集しています」を選択**
   (Sentry / PostHog に送信しているため「いいえ」は虚偽申告になる)

2. **収集するデータの種類**で以下の3つにチェックする。

   | データの種類 | 項目 | 根拠 |
   |---|---|---|
   | 診断 | クラッシュデータ | Sentry(クラッシュレポート) |
   | 使用状況データ | 製品の操作 | PostHog(定義済みイベント: オンボーディング完了、コーチ送信回数など) |
   | ID | ユーザーID | PostHog の distinct id として匿名の端末生成ID(deviceId)を使用 |

3. **各データ種類の詳細設定**(3種類とも同じ回答):
   - 利用目的: 「分析」のみにチェック(クラッシュデータは「アプリの機能」も可)
   - 「このデータはユーザーのIDに紐付けられていますか?」→ **いいえ(ユーザーに非紐付け)**
     (アカウント機能がなく、氏名・メール等と結び付かないランダムIDのみのため)
   - 「このデータをトラッキング目的で使用しますか?」→ **いいえ**
     (他社アプリ・Webサイトを横断した追跡、データブローカーへの共有は行っていない)

4. **収集していないと申告してよいもの**(実装上送信していない):
   - 連絡先情報(氏名・メールアドレス等)/ 位置情報 / 連絡先 / ユーザーコンテンツ(目標・記録・チャット本文はすべて端末内のみ。Sentry の `beforeSend`/`beforeBreadcrumb` でコーチAPIペイロードを除去済み)/ 閲覧履歴 / 購入 / 財務情報 / 健康とフィットネス / 身体的特徴

### 注意

- コーチAPI経由で Claude API にメッセージが送られるのは「AI応答の生成」という機能の実行であり、サーバー側で保存・収集していないため、Apple の定義上の「収集(collect)」には当たらない。ただし説明文・プライバシーポリシーには通信の事実を明記しておく(上記説明文に記載済み)。
- Sentry / PostHog の SDK を更新したり、オートキャプチャ・セッションリプレイ・`sendDefaultPii` などの設定を変えた場合は、このラベルの再確認が必要。
